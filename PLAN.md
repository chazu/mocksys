# Plan: contract-as-source, OpenAPI import, named/selectable examples

Goal: make it easy for agents to author mocks from (a) OpenAPI specs, (b) hand,
after reading API docs, (c) reading source. All three converge on a single
canonical **contract** that compiles down to a Mountebank imposter.

Decisions locked with the user:
- **Contract is canonical.** `contract.yaml` is edited; `imposter.json` is a build
  artifact regenerated by `compile`. `fault`/`parameterize` move onto the contract.
- **Schema primary, examples named & selectable.** Each operation keeps the formal
  JSON Schema; examples are named, multiple per response, and individually selected
  ("in play"). Selected examples become the stub's responses (Mountebank round-robins).
- **One scenario per operation** on OpenAPI import (operationId → scenario).
- Auto-recompile when `contract.yaml` is newer than `imposter.json` (invisible).
- Import defaults matchers to **method + path only** (anti-overfit, consistent with `doctor`).
- Migration is transparent: legacy imposter-only scenarios keep working; a contract
  is *lifted* from the imposter lazily the first time a contract-aware command needs one.

## On-disk layout (per scenario)

```
~/.mocksys/<service>/<name>/
  contract.yaml     # SOURCE OF TRUTH (string-keyed YAML; schemas/bodies kept faithfully)
  imposter.json     # BUILD ARTIFACT (compile output; never hand-edited)
  mock.yaml         # metadata (stub count, env, service) — unchanged
  recording.yaml    # record→freeze handoff — unchanged
```

## Contract format

```yaml
service: github
scenario: github/create-issue
origin: openapi            # openapi | recorded | authored (provenance)
fault: {}                  # optional transport fault overlay applied at compile
operations:
  - id: createIssue
    summary: Create an issue
    request:
      method: POST
      path: /repos/{owner}/{repo}/issues   # {param} → regex predicate on compile
      match: [method, path]
      bodySchema: {...}                     # optional, from spec
    responses:
      - status: 201
        headers: {Content-Type: application/json}
        bodySchema: {...}                   # PRIMARY: drives doctor/validate/sampling
        examples:
          happy:   {select: true,  body: {...}}
          minimal: {select: false, body: {...}}
      - status: 422
        examples:
          validation: {select: false, body: {...}}
```

Contract uses **string keys throughout** (like the imposter code already does), so
nested schemas/bodies round-trip faithfully through YAML without keyword mangling.

## Modules

- `store.cljs` (+): `write-contract!`/`read-contract` (raw YAML, no keywordize),
  `has-contract?`, `contract-stale?` (mtime: contract newer than imposter, or no imposter).
- `contract.cljs` (new):
  - `lower` contract → imposter (predicates from method/path incl. `{param}`→regex;
    responses = all `select:true` examples, each `{is {statusCode headers body}}`;
    if none selected, sample the first response's `bodySchema`; apply `fault` overlay).
  - `lift` imposter → contract (one operation per stub; example `recorded`).
  - `scaffold` minimal contract for `new`.
  - `validate` structural + lightweight example-vs-schema conformance (no ajv).
  - selection helpers: list examples, toggle `select`, `--only`.
  - `path->regex` (templated path → `^...[^/]+...$`), shared with `parameterize`.
- `openapi.cljs` (new): dereference spec (`@apidevtools/swagger-parser`), build one
  contract per operation, named examples from `examples`/`example`/sampler
  (`openapi-sampler`), synthesize + write the service template (servers→target,
  securitySchemes→redact headers).
- `core.cljs` (changes): new commands `import`, `compile`, `examples`, `use`, `new`,
  `validate`; `ensure-compiled!` (auto-recompile if stale) on run/env/inspect/doctor/pack;
  `ensure-contract!` (lift if missing) on contract-aware commands; `freeze` also lifts a
  contract; `add`/`fault`/`parameterize` rewritten to edit the contract + recompile;
  update `usage`, `prime`, dispatch.

## Command surface (new/changed)

```
mocksys import openapi <spec> [--service NAME] [--target URL]
mocksys compile <scenario>
mocksys new <scenario> [--service S]
mocksys validate <scenario>
mocksys examples <scenario> [--op ID]
mocksys use <scenario> --op ID --example NAME [--off] [--only]
mocksys add  <scenario> --request 'METHOD /path' [--status N] [--body F | --text S]   # now edits contract
mocksys fault <scenario> [--status N|--latency MS|--timeout|--drop-connection|--malformed-json|--clear]  # contract overlay
mocksys parameterize <scenario> --path '/a/{var}/b'   # templatizes the matching op, recompiles
```
